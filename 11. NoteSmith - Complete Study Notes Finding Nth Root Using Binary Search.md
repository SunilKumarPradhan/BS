## 📝 Problem Statement

### Definition
> **Find the Nth root of integer M**: Find a number `x` such that when multiplied `n` times, it equals `m`.
> 
> Mathematically: x^n = m

### Examples

**Example 1: Root Exists**
```
Input: n = 3, m = 27
Output: 3
Explanation: 3 × 3 × 3 = 27
```

**Example 2: Root Doesn't Exist**
```
Input: n = 4, m = 69
Output: -1
Explanation:
- 1^4 = 1 (too small)
- 2^4 = 16 (too small)
- 3^4 = 81 (too large, exceeded m)
- No integer satisfies the condition
```

### Key Observations
1. **Return exact root only** - No floor/ceiling values
2. **Return -1 if not found** - When no integer satisfies x^n = m
3. **Answer range**: Always between 1 and m
4. **Monotonic property**: If x^n > m, then (x+1)^n will also be > m

## 🐌 Brute Force Approach - Linear Search

### Concept
Try every number from 1 to m and check if raising it to power n gives m.

### Why This Works
- **Incremental checking**: Start from 1, go up until we find answer or exceed m
- **Early termination**: Stop when value exceeds m (no point checking further)

### Algorithm Steps
1. Loop from i = 1 to m
2. Calculate i^n
3. If i^n == m → return i
4. If i^n > m → break (answer doesn't exist)
5. If loop completes without finding → return -1

### Code Implementation

```python
def nth_root_brute_force(n, m):
    """
    Find nth root of m using linear search
    
    Args:
        n: The root to find (e.g., 3 for cube root)
        m: The number whose root we need
    
    Returns:
        The nth root if exists, else -1
    """
    # Try every number from 1 to m
    for i in range(1, m + 1):
        # Calculate i raised to power n
        power_value = calculate_power(i, n)
        
        # If exact match found
        if power_value == m:
            return i
        
        # If exceeded, no point checking further
        # (all subsequent numbers will also exceed)
        elif power_value > m:
            break
    
    # No root found
    return -1


def calculate_power(base, exponent):
    """
    Calculate base^exponent using simple multiplication
    
    Args:
        base: The number to multiply
        exponent: How many times to multiply
    
    Returns:
        base raised to power exponent
    """
    result = 1
    for _ in range(exponent):
        result *= base
    return result

print(nth_root_brute_force(4, 69))   # Output: -1
print(nth_root_brute_force(2, 16))   # Output: 4
```

### Time Complexity Analysis

**Overall Complexity: O(m × n)** or **O(m × log n)** depending on power calculation method

**Breakdown:**
- **Loop iterations**: O(m) - worst case we check all numbers from 1 to m
- **Power calculation per iteration**: 
  - Simple multiplication: O(n) - multiply n times
  - Binary exponentiation: O(log n) - halve the problem each time

**Space Complexity: O(1)** - Only using a few variables

## 🚀 Optimal Approach - Binary Search on Answers

### What is Binary Search on Answers?

> **Binary Search on Answers** is a technique where instead of searching in an array, we search in the **range of possible answers**.

### Key Insight
The answer space (1 to m) has a **monotonic property**:
- If x^n < m → answer might be on the right (try larger numbers)
- If x^n > m → answer is definitely on the left (all larger numbers will also exceed)
- If x^n = m → found the answer!

### Why Binary Search Works Here

**The Decision Pattern:**
```
Number:  1    2    3    4    5    6    7    ...    m
         ↓    ↓    ↓    ↓    ↓    ↓    ↓           ↓
Value:   <m   <m   <m   =m   >m   >m   >m         >m
         ←─────────→    ✓    ←──────────────────→
         Too small    Answer    Too large
```

When we check a number:
- **Too small** (x^n < m): Eliminate left half, search right
- **Too large** (x^n > m): Eliminate right half, search left
- **Perfect** (x^n = m): Found answer, return immediately

### Algorithm Steps

1. **Initialize search space**: low = 1, high = m
2. **While low ≤ high**:
   - Calculate mid = (low + high) / 2
   - Calculate mid^n
   - **If mid^n == m**: Return mid (found answer)
   - **If mid^n < m**: Search right half (low = mid + 1)
   - **If mid^n > m**: Search left half (high = mid - 1)
3. **If loop ends**: Return -1 (no answer exists)

## 💻 Implementation Details

### Basic Implementation (Without Overflow Handling)

```python
def nth_root_binary_search_basic(n, m):
    """
    Find nth root using binary search - BASIC VERSION
    WARNING: This has overflow issues for large numbers!
    
    Args:
        n: The root to find
        m: The number whose root we need
    
    Returns:
        The nth root if exists, else -1
    """
    # Step 1: Define search space
    low = 1
    high = m
    
    # Step 2: Binary search
    while low <= high:
        # Calculate middle point
        mid = (low + high) // 2
        
        # Calculate mid^n
        mid_power_n = power(mid, n)
        
        # Check three cases
        if mid_power_n == m:
            return mid  # Found exact answer
        elif mid_power_n < m:
            low = mid + 1  # Search right half
        else:
            high = mid - 1  # Search left half
    
    # No answer found
    return -1


def power(base, exponent):
    """
    Calculate base^exponent using simple loop
    WARNING: Can overflow for large values!
    
    Args:
        base: Number to raise
        exponent: Power to raise to
    
    Returns:
        base^exponent
    """
    result = 1
    for _ in range(exponent):
        result *= base
    return result
```

### Why Basic Implementation Fails

**Problem: Integer Overflow**

Consider:
```
n = 10, m = 10^9
mid = (1 + 10^9) / 2 ≈ 5 × 10^8
mid^10 = (5 × 10^8)^10 = 5^10 × 10^80
```

This number is **astronomically large** and causes overflow!

**What Happens:**
- Python handles big integers, but becomes very slow
- C++/Java: Integer overflow → wrong results
- Memory issues for extremely large calculations

### Production-Ready Implementation (With Overflow Handling)

```python
def nth_root_binary_search(n, m):
    """
    Find nth root using binary search with overflow protection
    
    This is the production-ready version that handles edge cases.
    
    Args:
        n: The root to find (positive integer)
        m: The number whose root we need (positive integer)
    
    Returns:
        The nth root if exists, else -1
    
    Example:
        >>> nth_root_binary_search(3, 27)
        3
        >>> nth_root_binary_search(4, 69)
        -1
    """
    # Edge case: m = 1 always has root 1
    if m == 1:
        return 1
    
    # Define search space
    low = 1
    high = m
    
    # Binary search
    while low <= high:
        mid = (low + high) // 2
        
        # Use comparison function instead of calculating full power
        comparison_result = compare_power(mid, n, m)
        
        if comparison_result == 1:
            # mid^n == m (found answer)
            return mid
        elif comparison_result == 0:
            # mid^n < m (search right)
            low = mid + 1
        else:
            # mid^n > m (search left)
            high = mid - 1
    
    # No exact root exists
    return -1


def compare_power(base, exponent, target):
    """
    Compare base^exponent with target WITHOUT calculating full power
    
    This prevents overflow by stopping multiplication early.
    
    Args:
        base: Number to raise
        exponent: Power to raise to
        target: Number to compare against
    
    Returns:
        1 if base^exponent == target
        0 if base^exponent < target
        2 if base^exponent > target
    
    Key Insight:
        Stop multiplying as soon as we exceed target!
        No need to calculate the full enormous number.
    """
    result = 1
    
    for i in range(exponent):
        result *= base
        
        # Critical: Check for overflow DURING multiplication
        if result > target:
            return 2  # Already exceeded, no need to continue
    
    # After all multiplications
    if result == target:
        return 1  # Exact match
    else:
        return 0  # Less than target
