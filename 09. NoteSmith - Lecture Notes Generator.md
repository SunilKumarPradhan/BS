## 📖 Problem Statement & Understanding

### 🎯 Problem Definition

> **Find a peak element in an array where a peak element is strictly greater than its neighbors.**

**Formal Definition:**
An element `arr[i]` is a peak if:
- `arr[i] > arr[i-1]` (greater than left neighbor)
- `arr[i] > arr[i+1]` (greater than right neighbor)

**Important Notes:**
- Use **strictly greater than** (not greater than or equal to)
- Array can have **multiple peaks** - return any one
- First and last elements have only one neighbor
- Assume `-∞` (minus infinity) exists before first element and after last element

## 🧩 Core Concepts - Peak Element Definition

### What is a Peak?

A peak is like a **mountain top** in the array landscape:

```
Array visualization:
        8 ← Peak (greater than 7 and 5)
       / \
      7   5
     /     \
    6       4
   /         \
  5           3
 /             \
1               2
```

### Mathematical Definition

For an element at index `i` to be a peak:

```
Peak Condition:
arr[i-1] < arr[i] > arr[i+1]
     ↑       ↑       ↑
   left   current  right
```

### Boundary Conditions

**First Element (index 0):**
```
-∞ < arr[0] > arr[1]
 ↑     ↑        ↑
left current  right
```

**Last Element (index n-1):**
```
arr[n-2] < arr[n-1] > -∞
   ↑         ↑         ↑
  left    current    right
```

## 🐌 Brute Force Approach

### Concept

**Strategy:** Linear scan through array, check each element if it satisfies peak conditions.

### Why Start with Brute Force?

> **Interview Tip:** Always present the brute force solution first. It shows:
> - You understand the problem completely
> - You can think of a working solution
> - You have a baseline to optimize from

### Algorithm Steps

1. **Iterate** from index 0 to n-1
2. For each index `i`:
   - Check if it's the **first element** (index 0)
   - Check if it's the **last element** (index n-1)
   - Check if `arr[i] > arr[i-1]` AND `arr[i] > arr[i+1]`
3. Return the first peak found

### Code Implementation

```python
def find_peak_brute_force(arr):
    """
    Brute force approach to find a peak element.
    
    Time Complexity: O(n) - single pass through array
    Space Complexity: O(1) - no extra space used
    
    Args:
        arr: List of integers
        
    Returns:
        Index of a peak element
    """
    n = len(arr)
    
    # Iterate through each element
    for i in range(n):
        # Check left condition
        # Either it's the first element (no left neighbor)
        # OR current element is greater than left neighbor
        left_condition = (i == 0) or (arr[i] > arr[i - 1])
        
        # Check right condition
        # Either it's the last element (no right neighbor)
        # OR current element is greater than right neighbor
        right_condition = (i == n - 1) or (arr[i] > arr[i + 1])
        
        # If both conditions satisfied, we found a peak
        if left_condition and right_condition:
            return i  # Return index of peak
    
    # This line should never execute if input is valid
    return -1
```

### Complexity Analysis

**Time Complexity: O(n)**
- Single pass through array
- In worst case, check all n elements
- Example: Peak at last position requires checking all elements

**Space Complexity: O(1)**
- Only using variables: `n`, `i`, `left_condition`, `right_condition`
- No additional data structures
- Space doesn't grow with input size

### Why Brute Force is Not Optimal

```
For array of size n = 1,000,000:
- Brute force: ~1,000,000 comparisons
- Binary search: ~20 comparisons (log₂(1,000,000) ≈ 20)

Speed difference: 50,000x faster!
```

## ⚡ Optimized Binary Search Solution

### 🤔 Why Binary Search?

**Key Observations that lead to Binary Search:**

1. **Search Operation**: We're searching for something (a peak)
2. **Array Property**: Array has sorted "tendencies" (increasing then decreasing sections)
3. **Time Constraint**: O(n) is not acceptable → need O(log n)
4. **Elimination Possible**: Can eliminate half the search space based on conditions

### When to Think Binary Search?

```
Checklist for Binary Search:
✓ Need to find/search something
✓ Working with array/list
✓ Array has some ordering property
✓ Can eliminate portions of search space
✓ Need better than O(n) complexity
```

### Core Insight: The Slope Strategy

**Key Concept:** If you're standing at any point in the array:
- **Increasing slope** (arr[mid] > arr[mid-1]): Peak guaranteed on the right
- **Decreasing slope** (arr[mid] < arr[mid-1]): Peak guaranteed on the left

```
Visualization:

Increasing Curve (Peak on Right):
        ? ← Peak must be here
       /
      /
   mid
    /
   /

Decreasing Curve (Peak on Left):
   ? ← Peak must be here
    \
     \
     mid
       \
        \
```

### Why This Works - Mathematical Proof

**Case 1: Mid on Increasing Slope**
```
If arr[mid] > arr[mid-1]:
- Elements are increasing towards right
- Either continues increasing to boundary (boundary becomes peak)
- Or starts decreasing (transition point is peak)
- Either way, peak exists on right side
```

**Case 2: Mid on Decreasing Slope**
```
If arr[mid] < arr[mid-1]:
- Elements were higher on left
- Either continues decreasing from start (first element is peak)
- Or was increasing before (transition point is peak)
- Either way, peak exists on left side
```

### Algorithm Steps

**Step 1: Handle Single Element**
```python
if n == 1:
    return 0  # Single element is always a peak
```

**Step 2: Check Boundaries Separately**
```python
# Check first element
if arr[0] > arr[1]:
    return 0
    
# Check last element
if arr[n-1] > arr[n-2]:
    return n-1
```

**Step 3: Binary Search on Middle Elements**
```python
low = 1          # Start from second element
high = n - 2     # End at second-last element

while low <= high:
    mid = (low + high) // 2
    
    # Check if mid is peak
    if arr[mid] > arr[mid-1] and arr[mid] > arr[mid+1]:
        return mid
    
    # If on increasing slope, go right
    elif arr[mid] > arr[mid-1]:
        low = mid + 1
    
    # If on decreasing slope, go left
    else:
        high = mid - 1
```
