## 🔍 What is Binary Search?

> **Binary Search** is a searching algorithm that efficiently finds a target element in a **sorted** search space by repeatedly dividing the search interval in half.

### Key Characteristics

**Critical Requirement**: The search space MUST be sorted
- Works on sorted arrays, sorted lists, or any sorted data structure
- Can be applied to problems beyond arrays (we'll see this in advanced topics)

**How It Works**:
1. Start with the entire search space
2. Find the middle element
3. Compare middle with target:
   - If equal → Found it!
   - If target is greater → Search right half
   - If target is smaller → Search left half
4. Repeat until found or search space exhausted

### Why Binary Search Exists

**Problem with Linear Search**:
```
Array: [3, 4, 6, 7, 9, 12, 16, 17]
Target: 17

Linear Search: Check 3 → 4 → 6 → 7 → 9 → 12 → 16 → 17 ✓
Steps: 8 (worst case = array length)
Time Complexity: O(n)
```

**Binary Search Solution**:
```
Same array and target
Binary Search: Check 7 → 12 → 16 → 17 ✓
Steps: 4
Time Complexity: O(log n)
```

**When to Use Binary Search**:
- ✅ Search space is sorted
- ✅ Need to find specific element
- ✅ Large datasets (efficiency matters)
- ✅ Repeated searches on same data

**When NOT to Use**:
- ❌ Unsorted data (sorting first might be costly)
- ❌ Small datasets (linear search is simpler)
- ❌ Frequently changing data (maintaining sorted order is expensive)

## 📚 Real-Life Example: Dictionary Search

### The Scenario
You have a physical dictionary with thousands of words arranged alphabetically (A to Z). You need to find the word "Raj".

### Approach 1: Linear Search (Inefficient)
```
Step 1: Open page 1 → Check all words → Not found
Step 2: Open page 2 → Check all words → Not found
Step 3: Open page 3 → Check all words → Not found
...
Continue until you find "Raj"

Problem: Could take hundreds of page turns!
```

### Approach 2: Binary Search (Efficient)

**Step 1: Initial State**
```
Dictionary: [A...Z] (entire dictionary)
Target: "Raj"
Action: Open dictionary in the middle
```

**Step 2: First Split**
```
Middle page shows: Words starting with "S"
Analysis: 
- "Raj" comes BEFORE "S" alphabetically
- Right half has: S, T, U, V, W, X, Y, Z
- Left half has: A, B, C, D, E, F, G, H, I, J, K, L, M, N, O, P, Q, R

Decision: Eliminate right half, search left half only
```

**Step 3: Second Split**
```
Take left half, split again
Middle shows: Words starting with "I"
Analysis:
- "Raj" comes AFTER "I"
- Eliminate left portion (A-I)
- Keep right portion (J-R)
```

**Step 4: Continue**
```
Keep splitting until you land on "R" section
Then find "Raj"
```

### Key Insight
Instead of checking every page (linear), we **eliminate half the dictionary** with each split. This is the power of binary search!

**Comparison**:
- Linear: ~1000 pages → 1000 checks
- Binary: ~1000 pages → ~10 checks (log₂(1000) ≈ 10)

## 💻 Coding Problem Example

### Problem Statement
Given a **sorted array** of unique integers and a target value, return the index of the target if found, otherwise return -1.

### Example Setup
```
Array:  [3, 4, 6, 7, 9, 12, 16, 17]
Index:   0  1  2  3  4   5   6   7
Target: 6
Output: 2 (because array[2] = 6)
```

### Step-by-Step Walkthrough

#### Example 1: Target = 6 (Element Exists)

**Initial Setup**
```
Array:  [3, 4, 6, 7, 9, 12, 16, 17]
Index:   0  1  2  3  4   5   6   7
         ↑                       ↑
        low                    high

Search Space: Entire array (indices 0 to 7)
```

**Iteration 1**
```
Calculate mid:
mid = (low + high) / 2 = (0 + 7) / 2 = 3.5 → 3 (integer division)

Array:  [3, 4, 6, 7, 9, 12, 16, 17]
Index:   0  1  2  3  4   5   6   7
         ↑        ↑               ↑
        low      mid            high

Compare: array[mid] = 7, target = 6
Analysis: 6 < 7, so target must be in LEFT half
Action: Move high to mid - 1

New Search Space:
Array:  [3, 4, 6, 7]
Index:   0  1  2  3
         ↑     ↑
        low  high
```

**Iteration 2**
```
Calculate mid:
mid = (0 + 2) / 2 = 1

Array:  [3, 4, 6]
Index:   0  1  2
         ↑  ↑  ↑
        low mid high

Compare: array[mid] = 4, target = 6
Analysis: 6 > 4, so target must be in RIGHT half
Action: Move low to mid + 1

New Search Space:
Array:  [6]
Index:   2
         ↑
      low/high
```

**Iteration 3**
```
Calculate mid:
mid = (2 + 2) / 2 = 2

Array:  [6]
Index:   2
         ↑
    low/mid/high

Compare: array[mid] = 6, target = 6
Result: FOUND! Return index 2
```

#### Example 2: Target = 13 (Element Does NOT Exist)

**Initial Setup**
```
Array:  [3, 4, 6, 7, 9, 12, 16, 17]
Index:   0  1  2  3  4   5   6   7
         ↑                       ↑
        low                    high
```

**Iteration 1**
```
mid = (0 + 7) / 2 = 3
array[3] = 7, target = 13
13 > 7 → Search RIGHT half
low = mid + 1 = 4

New Search Space: [9, 12, 16, 17]
Indices: 4, 5, 6, 7
```

**Iteration 2**
```
mid = (4 + 7) / 2 = 5
array[5] = 12, target = 13
13 > 12 → Search RIGHT half
low = mid + 1 = 6

New Search Space: [16, 17]
Indices: 6, 7
```

**Iteration 3**
```
mid = (6 + 7) / 2 = 6
array[6] = 16, target = 13
13 < 16 → Search LEFT half
high = mid - 1 = 5

New State:
low = 6, high = 5
```

**Termination**
```
Condition Check: low > high (6 > 5)
Search space exhausted!
Result: Element not found, return -1
```

### Visual Representation of Search Space Reduction

```
Target = 6:
[3, 4, 6, 7, 9, 12, 16, 17]  → Check 7
[3, 4, 6]                     → Check 4
[6]                           → Check 6 ✓ FOUND

Target = 13:
[3, 4, 6, 7, 9, 12, 16, 17]  → Check 7
[9, 12, 16, 17]               → Check 12
[16, 17]                      → Check 16
[]                            → Search space empty ✗ NOT FOUND
```

## 🔄 Iterative Implementation

### The Three Pointers Concept

Binary search uses three key pointers:

1. **`low`**: Start of current search space
2. **`high`**: End of current search space
3. **`mid`**: Middle of current search space

> **Search Space Rule**: Everything between `low` and `high` (inclusive) is your current search space.

### Pseudocode with Detailed Explanation

```python
function binarySearch(array, size, target):
    # Step 1: Initialize search space to entire array
    low = 0              # Start at first index
    high = size - 1      # End at last index
    
    # Step 2: Keep searching while search space exists
    while low <= high:   # Continue if search space is valid
        
        # Step 3: Find middle of current search space
        mid = (low + high) / 2  # Integer division
        
        # Step 4: Check if we found the target
        if array[mid] == target:
            return mid   # Found! Return the index
        
        # Step 5: Target is in right half
        elif target > array[mid]:
            low = mid + 1    # Eliminate left half
        
        # Step 6: Target is in left half
        else:  # target < array[mid]
            high = mid - 1   # Eliminate right half
    
    # Step 7: Search space exhausted, element not found
    return -1
```

### Line-by-Line Breakdown

**Line 1-2: Function Definition**
```python
function binarySearch(array, size, target):
```
- `array`: The sorted array to search in
- `size`: Length of the array
- `target`: The value we're looking for
- Returns: Index of target if found, -1 otherwise

**Line 3-4: Initialize Pointers**
```python
low = 0
high = size - 1
```
- `low = 0`: Start search from first element (index 0)
- `high = size - 1`: End search at last element
- Example: Array of size 8 → high = 7 (last valid index)

**Line 6: Loop Condition**
```python
while low <= high:
```
- **Why `<=` and not `<`?** 
  - When `low == high`, we have ONE element left to check
  - When `low > high`, search space is empty (crossed over)
- Loop continues as long as search space is valid

**Line 8: Calculate Middle**
```python
mid = (low + high) / 2
```
- Finds the middle index of current search space
- Uses integer division (rounds down)
- Example: `(0 + 7) / 2 = 3.5 → 3`

**Line 10-11: Found Target**
```python
if array[mid] == target:
    return mid
```
- Direct match found at middle position
- Immediately return the index
- No need to continue searching

**Line 13-14: Search Right Half**
```python
elif target > array[mid]:
    low = mid + 1
```
- Target is larger than middle element
- In sorted array, target must be to the RIGHT
- Move `low` pointer to `mid + 1` (eliminate left half including mid)
- New search space: `[mid+1, high]`

**Line 16-17: Search Left Half**
```python
else:
    high = mid - 1
```
- Only remaining case: `target < array[mid]`
- Target must be to the LEFT
- Move `high` pointer to `mid - 1` (eliminate right half including mid)
- New search space: `[low, mid-1]`

**Line 19: Not Found**
```python
return -1
```
- Loop exited without finding target
- Means `low > high` (search space exhausted)
- Return -1 to indicate "not found"

### Complete Working Example in Python

```python
def binary_search(array, size, target):
    """
    Performs binary search on a sorted array.
    
    Args:
        array: Sorted list of integers
        size: Length of the array
        target: Value to search for
    
    Returns:
        Index of target if found, -1 otherwise
    """
    low = 0
    high = size - 1
    
    while low <= high:
        # Calculate middle index
        mid = (low + high) // 2  # // is integer division in Python
        
        # Check if target is at mid
        if array[mid] == target:
            return mid
        
        # If target is greater, ignore left half
        elif target > array[mid]:
            low = mid + 1
        
        # If target is smaller, ignore right half
        else:
            high = mid - 1
    
    # Target not found
    return -1
