## 📚 Understanding Rotated Sorted Arrays

### What is a Rotated Sorted Array?

> A **rotated sorted array** is a sorted array that has been rotated around a pivot point.

**Original Sorted Array:**
```
[0, 1, 2, 4, 5, 6, 7]
```

**After Rotation at Index 4:**
```
[4, 5, 6, 7, 0, 1, 2]
         ↑
    Pivot Point
```

### Visual Representation

```
Original:  [0, 1, 2, 4, 5, 6, 7]
                      ↓ Rotate here
Rotated:   [4, 5, 6, 7 | 0, 1, 2]
            ←Sorted→   ←Sorted→
```

### Key Properties

1. **Two Sorted Portions**: The array contains two sorted subarrays
2. **Pivot Point**: The rotation point always contains the minimum element
3. **One Unsorted Half**: When you pick a mid point, one half will be sorted, the other won't be (unless you've crossed the pivot)

**Example Analysis:**
```python
Array: [4, 5, 6, 7, 0, 1, 2]
        L           M     H

Left Half:  [4, 5, 6, 7]  → Sorted (4 ≤ 7)
Right Half: [7, 0, 1, 2]  → Not Sorted (7 > 0)
```

## 🐌 Brute Force Approach

### The Obvious Solution

```python
def find_min_brute_force(arr):
    """
    Brute Force: Linear scan through array
    Time: O(n), Space: O(1)
    """
    minimum = float('inf')  # Start with infinity
    
    # Scan every element
    for num in arr:
        if num < minimum:
            minimum = num
    
    return minimum

# Example
arr = [4, 5, 6, 7, 0, 1, 2]
print(find_min_brute_force(arr))  # Output: 0
```

### Why This Works
- **Simple**: Just iterate and track the smallest value
- **Guaranteed**: Will always find the minimum

### Why Interviewers Won't Like It
- **Time Complexity**: O(n) - doesn't leverage the sorted property
- **Missed Opportunity**: The array has structure we're not using
- **Not Optimal**: We can do better with O(log n)

## 🔍 Binary Search Intuition

### Why Binary Search?

When you see **"sorted"** in a problem, think **binary search**. Even though the array is rotated, portions of it are still sorted.

### Binary Search Principle

> Binary search works by **eliminating half** of the search space in each iteration.

**Standard Binary Search:**
```
n → n/2 → n/4 → n/8 → ... → 1
```

### The Challenge

In a rotated array, we can't directly apply standard binary search because:
1. The array isn't fully sorted
2. We need to **identify which half is sorted** first
3. Then decide which half to eliminate

## 💡 Key Observations

### Observation 1: Identifying the Sorted Half

```python
Array: [4, 5, 6, 7, 0, 1, 2]
        L     M           H

# Check if left half is sorted
if arr[L] <= arr[M]:  # 4 <= 7 → True
    # Left half [4,5,6,7] is sorted
```

**Rule:**
- If `arr[low] <= arr[mid]` → **Left half is sorted**
- Otherwise → **Right half is sorted**

### Observation 2: Where is the Minimum?

**Critical Insight:**
> The **pivot point** (rotation point) always contains the minimum element.

**Why?**
```
Original: [0, 1, 2, 4, 5, 6, 7]
Rotated:  [4, 5, 6, 7, 0, 1, 2]
                      ↑
                   Minimum
```

The smallest element from the original array ends up at the rotation point.

### Observation 3: Sorted Half and Minimum

**Key Realization:**
> The sorted half **may or may not** contain the minimum.

**Case 1: Left sorted, minimum on right**
```python
[4, 5, 6, 7, 0, 1, 2]
 ←Sorted→   ↑
         Minimum here
```

**Case 2: Right sorted, minimum on left**
```python
[3, 4, 5, 1, 2]
         ↑  ←Sorted→
      Minimum
```

**Case 3: Right sorted, minimum in right**
```python
[7, 0, 1, 2, 3]
    ↑  ←Sorted→
  Minimum
```

**Case 4: Both sorted (crossed pivot)**
```python
[1, 2, 3, 4, 5]
 ↑
Minimum (entire array sorted)
```

### Observation 4: Strategy

**The Winning Strategy:**
1. Identify which half is sorted
2. **Pick the minimum from the sorted half** (it's always at the start)
3. **Eliminate that sorted half** (we've already taken its minimum)
4. Search in the other half

**Why this works:**
- The sorted half's minimum is at its leftmost position
- We capture it and eliminate that half
- The unsorted half contains the pivot (true minimum)
- We keep searching there

## 🛠️ Solution Development

### Step-by-Step Algorithm

**Setup:**
```python
low = 0
high = n - 1
answer = infinity  # Track minimum found so far
```

**Binary Search Loop:**
```
While low <= high:
    1. Calculate mid
    2. Check if left half is sorted
       - If yes: take arr[low] as minimum of that half
                 update answer
                 eliminate left half (low = mid + 1)
    3. Else (right half is sorted)
       - Take arr[mid] as minimum of right half
       - Update answer
       - Eliminate right half (high = mid - 1)
```

**Array:** `[4, 5, 6, 7, 0, 1, 2]`

```

├─ arr = [4, 5, 6, 7, 0, 1, 2]
│         L     M           H
├─ Left half [4,5,6,7] sorted? YES (4 <= 7)
├─ Minimum in left half = arr[0] = 4
├─ answer = min(∞, 4) = 4
└─ Eliminate left: low = 4

Iteration 2:
├─ low=4, high=6, mid=5
├─ arr = [4, 5, 6, 7, 0, 1, 2]
│                     L  M  H
├─ Left half [0,1] sorted? YES (0 <= 1)
├─ Minimum in left half = arr[4] = 0
├─ answer = min(4, 0) = 0

Iteration 3:
├─ low=6, high=6, mid=6

├─ Left half [2] sorted? YES
├─ Minimum = arr[6] = 2
```

## 💻 Code Implementation

### Basic Solution (Without Optimization)

```python
def findMin(arr):
    """
    Find minimum in rotated sorted array using binary search.
    
    Time Complexity: O(log n)
    Space Complexity: O(1)
    
    Args:
        arr: List of unique integers (rotated sorted array)
    
    Returns:
        int: Minimum element in the array
    """
    # Initialize pointers
    low = 0
    high = len(arr) - 1
    answer = float('inf')  # Track minimum found
    
    # Binary search
    while low <= high:
        # Calculate mid point
        mid = low + (high - low) // 2  # Prevents overflow
        
        # Check if LEFT half is sorted
        if arr[low] <= arr[mid]:
            # Left half is sorted: [low...mid]
            # Minimum of sorted portion is at the start
            answer = min(answer, arr[low])
            
            # Eliminate left half (already captured its minimum)
            low = mid + 1
        
        else:
            # RIGHT half is sorted: [mid...high]
            # Minimum of right sorted portion is at mid
            answer = min(answer, arr[mid])
            
            # Eliminate right half
            high = mid - 1
    
    return answer


# Test Cases
print(findMin([4, 5, 6, 7, 0, 1, 2]))  # Output: 0
print(findMin([3, 4, 5, 1, 2]))        # Output: 1
print(findMin([1, 2]))                 # Output: 1
print(findMin([2, 1]))                 # Output: 1
print(findMin([1]))                    # Output: 1
```
