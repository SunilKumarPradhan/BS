## 🔍 Problem 1: First and Last Occurrence

### Problem Statement

> **Given**: A sorted array with duplicates and a target element X
> 
> **Find**: The first and last index where X appears
> 
> **Return**: A pair of indices `[first_index, last_index]`. If X doesn't exist, return `[-1, -1]`

### Examples

**Example 1:**
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
X = 8
Output: [3, 5]
Explanation: 8 first appears at index 3 and last appears at index 5
```

**Example 2:**
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
X = 10
Output: [-1, -1]
Explanation: 10 doesn't exist in the array
```

**Example 3:**
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
X = 11
Output: [6, 6]
Explanation: 11 appears only once at index 6
```

## 🐌 Brute Force Approach

### Concept
Perform a **linear scan** through the entire array, tracking the first and last occurrence as we encounter the target element.

### Algorithm Logic

1. **Initialize** `first = -1` and `last = -1`
2. **Iterate** through array from index 0 to n-1
3. **For each element**:
   - If `array[i] == X`:
     - If `first == -1` (first time finding X), set `first = i`
     - Always update `last = i` (keeps updating to rightmost occurrence)
4. **Return** `[first, last]`

### Why This Works
- Since the array is sorted, all occurrences of X are consecutive
- The first time we find X is the leftmost occurrence
- The last time we find X is the rightmost occurrence

### Code Implementation

```python
def first_last_occurrence_brute(arr, n, x):
    """
    Find first and last occurrence using linear search
    
    Args:
        arr: Sorted array with possible duplicates
        n: Length of array
        x: Target element to find
    
    Returns:
        List containing [first_index, last_index]
    """
    # Initialize both positions as -1 (not found)
    first = -1
    last = -1
    
    # Linear iteration through entire array
    for i in range(n):
        # Check if current element matches target
        if arr[i] == x:
            # If this is first occurrence (first is still -1)
            if first == -1:
                first = i  # Store the first occurrence index
            
            # Always update last (will keep updating till rightmost)
            last = i
    
    # Return the pair of indices
    return [first, last]


# Test Cases
print(first_last_occurrence_brute([2, 4, 6, 8, 8, 8, 11, 13], 8, 8))  # [3, 5]
print(first_last_occurrence_brute([2, 4, 6, 8, 8, 8, 11, 13], 8, 10)) # [-1, -1]
print(first_last_occurrence_brute([2, 4, 6, 8, 8, 8, 11, 13], 8, 11)) # [6, 6]
print(first_last_occurrence_brute([5, 5, 5, 5], 4, 5))                # [0, 3]
```

### Complexity Analysis

| Metric | Complexity | Explanation |
|--------|------------|-------------|
| **Time** | O(n) | We traverse entire array once in worst case |
| **Space** | O(1) | Only using two variables (first, last) |

### When to Use
- Array is very small (n < 100)
- You need to understand the basic approach first
- Interviewer asks for the simplest solution first

### Limitations
- **Not optimal** for large datasets
- **Doesn't utilize** the sorted property of array
- Interviewer will ask for optimization

## ⚡ Optimal Approach Using Lower & Upper Bound

### Concept: Understanding Lower and Upper Bound

Before diving into the solution, let's understand two critical binary search concepts:

#### Lower Bound
> **Definition**: The smallest index where `array[index] >= X`
> 
> In other words: First position where you can insert X while maintaining sorted order

**Visual Example:**
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
       [0, 1, 2, 3, 4, 5, 6,  7]

Lower Bound of 8:
- Looking for first element >= 8
- Points to index 3 (first 8)
```

#### Upper Bound
> **Definition**: The smallest index where `array[index] > X`
> 
> In other words: First position where element is strictly greater than X

**Visual Example:**
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
       [0, 1, 2, 3, 4, 5, 6,  7]

Upper Bound of 8:
- Looking for first element > 8
- Points to index 6 (11)
```

### The Key Insight

For finding first and last occurrence:
- **First Occurrence** = Lower Bound of X
- **Last Occurrence** = Upper Bound of X - 1

### Why This Works

```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
              ↑           ↑
              |           |
        Lower Bound    Upper Bound
         (index 3)      (index 6)
         
First Occurrence = 3
Last Occurrence = 6 - 1 = 5 ✓
```

### Edge Cases to Handle

#### Case 1: Element Doesn't Exist
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
X = 10

Lower Bound of 10:
- Points to index 6 (11, first element >= 10)
- But arr[6] = 11 ≠ 10
- Element doesn't exist!
```

#### Case 2: Element Greater Than All
```
Array: [2, 4, 6, 8, 8, 8, 11, 13]
X = 14

Lower Bound of 14:
- Points to index 8 (hypothetical, beyond array)
- lb == n means element doesn't exist
```

### Algorithm Steps

1. **Find Lower Bound** of X → gives potential first occurrence
2. **Check Validity**:
   - If `lb == n` → X doesn't exist
   - If `arr[lb] != X` → X doesn't exist
3. **Find Upper Bound** of X → gives index after last occurrence
4. **Calculate Last** = Upper Bound - 1
5. **Return** `[lb, ub - 1]`

### Code Implementation

```python
def lower_bound(arr, n, x):
    """
    Find the smallest index where arr[index] >= x
    
    Args:
        arr: Sorted array
        n: Length of array
        x: Target element
    
    Returns:
        Index of lower bound (can be n if all elements < x)
    """
    low = 0
    high = n - 1
    ans = n  # Default: if all elements < x, return n
    
    while low <= high:
        mid = (low + high) // 2
        
        # If current element >= x, this could be answer
        if arr[mid] >= x:
            ans = mid      # Store potential answer
            high = mid - 1 # Look for smaller index on left
        else:
            # Current element < x, search right
            low = mid + 1
    
    return ans


def upper_bound(arr, n, x):
    """
    Find the smallest index where arr[index] > x
    
    Args:
        arr: Sorted array
        n: Length of array
        x: Target element
    
    Returns:
        Index of upper bound (can be n if all elements <= x)
    """
    low = 0
    high = n - 1
    ans = n  # Default: if all elements <= x, return n
    
    while low <= high:
        mid = (low + high) // 2
        
        # If current element > x, this could be answer
        if arr[mid] > x:
            ans = mid      # Store potential answer
            high = mid - 1 # Look for smaller index on left
        else:
            # Current element <= x, search right
            low = mid + 1
    
    return ans


def first_last_occurrence_optimal_v1(arr, n, x):
    """
    Find first and last occurrence using lower and upper bound
    
    Args:
        arr: Sorted array with possible duplicates
        n: Length of array
        x: Target element
    
    Returns:
        List containing [first_index, last_index] or [-1, -1]
    """
    # Find lower bound (potential first occurrence)
    lb = lower_bound(arr, n, x)
    
    # Edge Case 1: Lower bound points beyond array
    # This means all elements are less than x
    if lb == n:
        return [-1, -1]
    
    # Edge Case 2: Element at lower bound is not x
    # This means x doesn't exist in array
    if arr[lb] != x:
        return [-1, -1]
    
    # Find upper bound (first element > x)
    ub = upper_bound(arr, n, x)
    
    # First occurrence = lower bound
    # Last occurrence = upper bound - 1
    return [lb, ub - 1]


# Test Cases
arr = [2, 4, 6, 8, 8, 8, 11, 13]
print(first_last_occurrence_optimal_v1(arr, 8, 8))   # [3, 5]
print(first_last_occurrence_optimal_v1(arr, 8, 10))  # [-1, -1]
print(first_last_occurrence_optimal_v1(arr, 8, 11))  # [6, 6]
print(first_last_occurrence_optimal_v1(arr, 8, 14))  # [-1, -1]
print(first_last_occurrence_optimal_v1([5, 5, 5], 3, 5))  # [0, 2]
```

### Dry Run Example

**Array**: `[2, 4, 6, 8, 8, 8, 11, 13]`, **X = 8**

#### Finding Lower Bound of 8:

```
Iteration 1:
low = 0, high = 7
mid = (0 + 7) // 2 = 3
arr[3] = 8 >= 8 ✓
ans = 3
high = 2 (search left for smaller index)

Iteration 2:
low = 0, high = 2
mid = (0 + 2) // 2 = 1
arr[1] = 4 < 8
low = 2 (search right)

Iteration 3:
low = 2, high = 2
mid = (2 + 2) // 2 = 2
arr[2] = 6 < 8
low = 3 (search right)

low > high → Stop
Lower Bound = 3
```

#### Finding Upper Bound of 8:

```
Iteration 1:
low = 0, high = 7
mid = 3
arr[3] = 8 NOT > 8
low = 4 (search right)

Iteration 2:
low = 4, high = 7
mid = 5
arr[5] = 8 NOT > 8
low = 6

Iteration 3:
low = 6, high = 7
mid = 6
arr[6] = 11 > 8 ✓
ans = 6
high = 5

low > high → Stop
Upper Bound = 6
```

**Result**: `[3, 6-1]` = `[3, 5]` ✓

### Complexity Analysis

| Metric | Complexity | Explanation |
|--------|------------|-------------|
| **Time** | O(log n) | Two binary searches: O(log n) + O(log n) = O(2 log n) ≈ O(log n) |
| **Space** | O(1) | Only using constant extra variables |

### When to Use
- You already know lower/upper bound concepts
- Interviewer is familiar with these concepts
- You want to reuse pre-written functions
- Code readability is prioritized

## 🎯 Optimal Approach Using Pure Binary Search

### Why This Approach?

Some interviewers may:
- Not be familiar with lower/upper bound terminology
- Want to see your understanding of binary search fundamentals
- Ask you to avoid "pre-built" concepts

This approach uses **two separate binary searches**:
1. One to find the **first occurrence**
2. One to find the **last occurrence**

### Finding First Occurrence

#### Core Idea
When we find the target element, we **don't stop** — we continue searching **left** to find an earlier occurrence.
