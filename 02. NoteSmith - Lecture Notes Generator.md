## 🔍 Lower Bound

### Definition

> **Lower Bound**: The smallest index in a sorted array where the element at that index is **greater than or equal to** the given target value X.

**Key Points:**
- Array **must be sorted**
- Returns an **index**, not a value
- If no element satisfies the condition, returns array size (hypothetical next index)
- Uses **≥** (greater than or equal to) comparison

### 📊 Understanding Through Examples

Let's use this sorted array: `[1, 2, 3, 5, 8, 15, 19]`

**Indices:**
```
Index:  0  1  2  3  4   5   6
Array: [1, 2, 3, 5, 8, 15, 19]
```

#### Example 1: X = 8
- **Question**: Find smallest index where element ≥ 8
- **Analysis**: 
  - Index 0: 1 < 8 ❌
  - Index 1: 2 < 8 ❌
  - Index 2: 3 < 8 ❌
  - Index 3: 5 < 8 ❌
  - Index 4: 8 ≥ 8 ✅ **FIRST MATCH**
- **Answer**: Index **4**

#### Example 2: X = 9
- **Question**: Find smallest index where element ≥ 9
- **Analysis**:
  - Indices 0-4: All elements < 9 ❌
  - Index 5: 15 ≥ 9 ✅ **FIRST MATCH**
- **Answer**: Index **5**

#### Example 3: X = 16
- **Question**: Find smallest index where element ≥ 16
- **Analysis**:
  - Indices 0-5: All elements < 16 ❌
  - Index 6: 19 ≥ 16 ✅ **FIRST MATCH**
- **Answer**: Index **6**

#### Example 4: X = 20
- **Question**: Find smallest index where element ≥ 20
- **Analysis**:
  - All elements < 20 ❌
  - No element satisfies condition
- **Answer**: Index **7** (array size, hypothetical index)

#### Example 5: X = 1
- **Question**: Find smallest index where element ≥ 1
- **Analysis**:
  - Index 0: 1 ≥ 1 ✅ **FIRST MATCH**
- **Answer**: Index **0**

### 🚶 Naive Approach (Linear Search)

**Concept**: Iterate through array from left to right until finding first element ≥ X

```python
def lower_bound_linear(arr, n, x):
    """
    Find lower bound using linear search
    
    Args:
        arr: Sorted array
        n: Size of array
        x: Target value
    
    Returns:
        Smallest index where arr[index] >= x
    """
    # Iterate through each element
    for i in range(n):
        # Check if current element satisfies condition
        if arr[i] >= x:
            return i  # Found first element >= x
    
    # No element found, return hypothetical index
    return n

# Example usage
arr = [1, 2, 3, 5, 8, 15, 19]
n = len(arr)

print(lower_bound_linear(arr, n, 8))   # Output: 4
print(lower_bound_linear(arr, n, 9))   # Output: 5
print(lower_bound_linear(arr, n, 20))  # Output: 7 (array size)
```

**Time Complexity**: O(n) - Must potentially check every element  
**Space Complexity**: O(1) - No extra space used

**Why This Isn't Optimal:**
- Doesn't utilize the **sorted** property of array
- Wastes time checking elements we know are smaller
- For large arrays (millions of elements), very slow

### ⚡ Optimized Approach (Binary Search)

**Why Binary Search?**
- Array is **sorted** → Can eliminate half the search space each iteration
- Looking for a **specific position** → Perfect use case for binary search
- Reduces time from O(n) to O(log n)

**Core Insight:**
- If `arr[mid] >= x`: This **might be** our answer, but check left for smaller index
- If `arr[mid] < x`: This **cannot be** our answer, search right for larger values

### 🎬 Detailed Binary Search Walkthrough

**Array**: `[1, 2, 3, 5, 7, 8, 10, 15, 19]`  
**Target (X)**: 9

**Initial Setup:**
```
Index:  0  1  2  3  4  5   6   7   8
Array: [1, 2, 3, 5, 7, 8, 10, 15, 19]
        ↑                          ↑
       low                        high
       
answer = 9 (array size, default if nothing found)
```

#### Iteration 1:
```python
low = 0, high = 8
mid = (0 + 8) // 2 = 4

Index:  0  1  2  3  4  5   6   7   8
Array: [1, 2, 3, 5, 7, 8, 10, 15, 19]
        ↑           ↑               ↑
       low         mid             high
```

**Check**: `arr[4] = 7`, is `7 >= 9`? **NO**
- 7 is too small
- Everything to the left is also too small (sorted array)
- **Action**: Search right half
- **Update**: `low = mid + 1 = 5`

#### Iteration 2:
```python
low = 5, high = 8
mid = (5 + 8) // 2 = 6

Index:  0  1  2  3  4  5   6   7   8
Array: [1, 2, 3, 5, 7, 8, 10, 15, 19]
                       ↑   ↑       ↑
                      low mid    high
```

**Check**: `arr[6] = 10`, is `10 >= 9`? **YES** ✅
- 10 satisfies our condition
- But we need the **smallest** index
- There might be a smaller index on the left
- **Action**: Store this as potential answer, search left
- **Update**: `answer = 6`, `high = mid - 1 = 5`

#### Iteration 3:
```python
low = 5, high = 5
mid = (5 + 5) // 2 = 5

Index:  0  1  2  3  4  5   6   7   8
Array: [1, 2, 3, 5, 7, 8, 10, 15, 19]
                       ↑
                   low/mid/high
```

**Check**: `arr[5] = 8`, is `8 >= 9`? **NO**
- 8 is too small
- **Action**: Search right
- **Update**: `low = mid + 1 = 6`

#### Loop Termination:
```python
low = 6, high = 5
# low > high, loop ends
```

**Final Answer**: `answer = 6` (index where 10 is located)

### 💻 Complete Implementation

```python
def lower_bound(arr, n, x):
    """
    Find the smallest index where arr[index] >= x using binary search.
    
    Algorithm:
    1. Initialize search space: low=0, high=n-1
    2. Set answer to n (default if no element found)
    3. While search space exists:
       - Calculate mid point
       - If arr[mid] >= x: potential answer, search left for smaller index
       - Else: search right for larger values
    4. Return the stored answer
    
    Args:
        arr (list): Sorted array of integers
        n (int): Size of the array
        x (int): Target value to find lower bound for
    
    Returns:
        int: Smallest index where arr[index] >= x, or n if not found
    
    Time Complexity: O(log n)
    Space Complexity: O(1)
    """
    # Initialize binary search boundaries
    low = 0           # Start of search space
    high = n - 1      # End of search space
    
    # Default answer: hypothetical index if no element >= x exists
    # This handles edge case where all elements < x
    answer = n
    
    # Binary search loop
    while low <= high:
        # Calculate middle index
        # Using (low + high) // 2 to avoid integer overflow in other languages
        mid = (low + high) // 2
        
        # Check if element at mid satisfies our condition
        if arr[mid] >= x:
            # This MIGHT be our answer (element >= x found)
            # But we need the SMALLEST index, so check left half
            answer = mid          # Store current index as potential answer
            high = mid - 1        # Search in left half for smaller index
        else:
            # arr[mid] < x
            # This cannot be our answer
            # All elements to the left are also smaller (sorted array)
            # So search in right half for larger values
            low = mid + 1
    
    # Return the smallest index found where arr[index] >= x
    return answer


# ============= EXAMPLE USAGE =============

# Example 1: Basic usage
arr1 = [1, 2, 3, 5, 8, 15, 19]
n1 = len(arr1)

print("Array:", arr1)
print(f"Lower bound of 8: {lower_bound(arr1, n1, 8)}")   # Output: 4
print(f"Lower bound of 9: {lower_bound(arr1, n1, 9)}")   # Output: 5
print(f"Lower bound of 16: {lower_bound(arr1, n1, 16)}") # Output: 6
print(f"Lower bound of 20: {lower_bound(arr1, n1, 20)}") # Output: 7 (n)
print(f"Lower bound of 1: {lower_bound(arr1, n1, 1)}")   # Output: 0

# Example 2: With duplicates
arr2 = [1, 2, 3, 3, 7, 8, 9, 9, 9, 11]
n2 = len(arr2)

print("\nArray with duplicates:", arr2)
print(f"Lower bound of 3: {lower_bound(arr2, n2, 3)}")   # Output: 2 (first 3)
print(f"Lower bound of 9: {lower_bound(arr2, n2, 9)}")   # Output: 6 (first 9)

# Example 3: Edge cases
arr3 = [5, 10, 15, 20]
n3 = len(arr3)

print("\nEdge cases:")
print(f"Lower bound of 0: {lower_bound(arr3, n3, 0)}")   # Output: 0 (before all)
print(f"Lower bound of 25: {lower_bound(arr3, n3, 25)}") # Output: 4 (after all)
print(f"Lower bound of 10: {lower_bound(arr3, n3, 10)}") # Output: 1 (exact match)
```
