## 📋 Problem Statement

**Given**: A sorted array that has been rotated an unknown number of times (contains unique elements)

**Find**: The number of times the array has been rotated

### Example 1:
```
Original sorted array: [1, 2, 3, 4, 5, 7]
Rotated array:         [3, 4, 5, 7, 1, 2]
Answer: 4 (rotated 4 times)
```

**Explanation**: The array was rotated 4 positions to the right. The minimum element (1) is now at index 4.

### Example 2:
```
Original sorted array: [1, 2, 4, 5, 7]
Rotated array:         [1, 2, 4, 5, 7]
Answer: 0 (not rotated)
```

### Example 3:
```
Original sorted array: [1, 2, 3, 4, 5]
Rotated array:         [5, 1, 2, 3, 4]
Answer: 1 (rotated 1 time)
```

## 🔄 Core Concept: Understanding Array Rotation

### What is Array Rotation?

Array rotation means moving elements from one end to another. A **right rotation** moves elements to the right, wrapping around.

```
Original: [1, 2, 3, 4, 5]

After 1 rotation:  [5, 1, 2, 3, 4]
After 2 rotations: [4, 5, 1, 2, 3]
After 3 rotations: [3, 4, 5, 1, 2]
```

### Visual Representation

```
Index:  0  1  2  3  4  5
        ↓  ↓  ↓  ↓  ↓  ↓
Array: [3, 4, 5, 7, 1, 2]
                    ↑
                 Minimum
                (Index = 4)
```

The minimum element's index tells us the rotation count!

### Why Does This Work?

In a sorted array, the minimum element is at index 0. When rotated `k` times, the minimum element moves to index `k`. Therefore:

> **Rotation Count = Index of Minimum Element**

## 💡 The Key Insight

### The Breakthrough Realization

If you can solve **"Find Minimum in Rotated Sorted Array"**, you can solve this problem by simply tracking the **index** of the minimum element!

### The Connection

| Problem | What to Return |
|---------|----------------|
| Find Minimum in Rotated Sorted Array | The minimum **value** |
| Find Rotation Count | The **index** of minimum value |

### Why This Works

1. **Sorted Array Property**: In a sorted array, minimum is at index 0
2. **Rotation Effect**: Rotation shifts the minimum to index `k` (where `k` = rotation count)
3. **Direct Mapping**: Index of minimum = Number of rotations

## 🛠️ Solution Approach

### Strategy Overview

We'll use **modified binary search** to find the minimum element while tracking its index.

### Algorithm Steps

1. **Initialize** two pointers: `low = 0`, `high = n-1`
2. **Track** the minimum value and its index
3. **Binary Search Loop**:
   - Calculate `mid`
   - Check if left half is sorted
   - Update minimum and index if needed
   - Eliminate one half
4. **Return** the index of minimum

### Decision Tree

```
                    [3, 4, 5, 7, 1, 2]
                           ↓
                    Is left sorted?
                    /              \
                  YES               NO
                   ↓                 ↓
            Update min if       Update min if
            arr[low] < min      arr[mid] < min
                   ↓                 ↓
            Search right        Search left
```

### Why Binary Search?

- **Linear Search**: O(n) - Check every element
- **Binary Search**: O(log n) - Eliminate half each time
- **Efficiency**: For array of 1 million elements, binary search needs only ~20 comparisons vs 1 million

## 💻 Code Implementation

### Complete Solution with Detailed Explanation

```python
def findKRotation(arr):
    """
    Find the number of times a sorted array has been rotated.
    
    Args:
        arr: List of unique integers (rotated sorted array)
    
    Returns:
        int: Number of rotations (index of minimum element)
    
    Time Complexity: O(log n)
    Space Complexity: O(1)
    """
    # Step 1: Initialize binary search boundaries
    low = 0
    high = len(arr) - 1
    
    # Step 2: Initialize answer variables
    # We track both the minimum value and its index
    answer = float('inf')  # Minimum value found so far
    index = -1              # Index of minimum value
    
    # Step 3: Binary search loop
    while low <= high:
        # Calculate middle index
        mid = (low + high) // 2
        
        # OPTIMIZATION: If entire remaining array is sorted
        # The minimum must be at 'low'
        if arr[low] <= arr[high]:
            if arr[low] < answer:
                answer = arr[low]
                index = low
            break
        
        # Step 4: Check if LEFT half is sorted
        if arr[low] <= arr[mid]:
            # Left half is sorted
            # Minimum of left half is arr[low]
            if arr[low] < answer:
                answer = arr[low]
                index = low
            
            # Since left is sorted, minimum must be in right half
            low = mid + 1
        
        # Step 5: RIGHT half is sorted
        else:
            # Right half is sorted
            # Minimum of right half is arr[mid]
            if arr[mid] < answer:
                answer = arr[mid]
                index = mid
            
            # Since right is sorted, minimum must be in left half
            high = mid - 1
    
    # Step 6: Return the index (rotation count)
    return index
```

### Line-by-Line Breakdown

#### Initialization Phase
```python
low = 0
high = len(arr) - 1
```
- `low`: Start of search space (leftmost index)
- `high`: End of search space (rightmost index)
- **Why?** Binary search requires boundaries to divide the array

```python
answer = float('inf')
index = -1
```
- `answer`: Stores the smallest value found (initialized to infinity)
- `index`: Stores position of smallest value (initialized to -1 as sentinel)
- **Why float('inf')?** Ensures any array element will be smaller in first comparison

#### Main Loop
```python
while low <= high:
```
- **Condition**: Continue while search space is valid
- **Termination**: When `low > high`, we've searched everything

```python
mid = (low + high) // 2
```
- **Purpose**: Find middle element to divide array
- **`//` operator**: Integer division (no decimals)
- **Example**: If `low=0, high=5`, then `mid = (0+5)//2 = 2`

#### Optimization Check
```python
if arr[low] <= arr[high]:
    if arr[low] < answer:
        answer = arr[low]
        index = low
    break
```
- **When this happens**: Remaining array is completely sorted
- **Why break?** No need to continue - minimum is at `low`
- **Example**: `[1, 2, 3, 4, 5]` - already sorted, minimum at index 0

#### Left Half Sorted
```python
if arr[low] <= arr[mid]:
```
- **Check**: Is left portion sorted?
- **How to tell**: If first element ≤ middle element, it's sorted
- **Example**: `[3, 4, 5, 7, 1, 2]` with `mid=2` → `arr[0]=3 ≤ arr[2]=5` ✓

```python
if arr[low] < answer:
    answer = arr[low]
    index = low
```
- **Update minimum**: If left's minimum (arr[low]) is smaller than current answer
- **Track index**: Store where we found this minimum

```python
low = mid + 1
```
- **Eliminate left half**: Since it's sorted, minimum can't be there
- **Search right**: Move to unsorted portion

#### Right Half Sorted
```python
else:
    if arr[mid] < answer:
        answer = arr[mid]
        index = mid
    high = mid - 1
```
- **Logic**: If left isn't sorted, right must be sorted
- **Update**: Check if `arr[mid]` (right's minimum) is smaller
- **Eliminate right**: Search in left unsorted portion

### Example Walkthrough

Let's trace through `arr = [3, 4, 5, 7, 1, 2]`:

```
Initial: low=0, high=5, answer=∞, index=-1

Iteration 1:
  mid = (0+5)//2 = 2
  arr[low]=3, arr[high]=2 → Not fully sorted
  arr[low]=3 <= arr[mid]=5 → Left sorted
  arr[low]=3 < answer=∞ → Update: answer=3, index=0
  Search right: low=3

Iteration 2:
  low=3, high=5
  mid = (3+5)//2 = 4
  arr[low]=7, arr[high]=2 → Not fully sorted
  arr[low]=7 > arr[mid]=1 → Right sorted
  arr[mid]=1 < answer=3 → Update: answer=1, index=4
  Search left: high=3

Iteration 3:
  low=3, high=3
  mid = (3+3)//2 = 3
  arr[low]=7, arr[high]=7 → Fully sorted
  arr[low]=7 > answer=1 → No update
  Break

Return: index = 4 ✓
```

### Alternative Compact Version

```python
def findKRotation(arr):
    low, high = 0, len(arr) - 1
    min_val, min_idx = float('inf'), -1
    
    while low <= high:
        mid = (low + high) // 2
        
        # Optimization for sorted array
        if arr[low] <= arr[high]:
            if arr[low] < min_val:
                min_idx = low
            break
        
        # Left sorted
        if arr[low] <= arr[mid]:
            if arr[low] < min_val:
                min_val, min_idx = arr[low], low
            low = mid + 1
        # Right sorted
        else:
            if arr[mid] < min_val:
                min_val, min_idx = arr[mid], mid
            high = mid - 1
    
    return min_idx
```

## 🔁 Handling Duplicates

### The Problem with Duplicates

When the array contains duplicate elements, our simple check `arr[low] <= arr[mid]` becomes ambiguous.

### Example of Ambiguity

```
Array: [2, 2, 2, 0, 1, 2]
       low=0, mid=2, high=5

arr[low] = 2
arr[mid] = 2
arr[high] = 2

Which half is sorted? Can't tell!
```

### Solution: Shrink Search Space

When `arr[low] == arr[mid] == arr[high]`, we can't determine which half to eliminate. Instead:

```python
def findKRotationWithDuplicates(arr):
    low, high = 0, len(arr) - 1
    min_val, min_idx = float('inf'), -1
    
    while low <= high:
        mid = (low + high) // 2
        
        # Handle duplicates at boundaries
        if arr[low] == arr[mid] == arr[high]:
            # Check low
            if arr[low] < min_val:
                min_val, min_idx = arr[low], low
            low += 1  # Shrink from left
            
            # Check high
            if arr[high] < min_val:
                min_val, min_idx = arr[high], high
            high -= 1  # Shrink from right
            continue
        
        # Optimization for sorted array
        if arr[low] <= arr[high]:
            if arr[low] < min_val:
                min_val, min_idx = arr[low], low
            break
        
        # Left sorted
        if arr[low] <= arr[mid]:
            if arr[low] < min_val:
                min_val, min_idx = arr[low], low
            low = mid + 1
        # Right sorted
        else:
            if arr[mid] < min_val:
                min_val, min_idx = arr[mid], mid
            high = mid - 1
    
    return min_idx
```

### Key Changes Explained

```python
if arr[low] == arr[mid] == arr[high]:
    if arr[low] < min_val:
        min_val, min_idx = arr[low], low
    low += 1
    
    if arr[high] < min_val:
        min_val, min_idx = arr[high], high
    high -= 1
    continue
```

- **When**: All three positions have same value
- **Action**: Check both boundaries, then shrink by 1 from each side
- **Why**: We can't eliminate half, so we eliminate just 1 element at a time
- **Trade-off**: Worst case becomes O(n) instead of O(log n)

### Complexity Impact

| Scenario | Without Duplicates | With Duplicates |
|----------|-------------------|-----------------|
| Best Case | O(log n) | O(log n) |
| Average Case | O(log n) | O(log n) |
| Worst Case | O(log n) | O(n) |

**Worst case example**: `[2, 2, 2, 2, 2, 2, 0, 2]` - Must check almost all elements
