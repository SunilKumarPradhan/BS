## 🧩 Problem Understanding

### The Core Concept

> **All three problems ask the same fundamental question:**
> "How do we split an array into K contiguous subarrays such that the maximum sum among all subarrays is minimized?"

This is called the **Min-Max pattern** or **Minimize the Maximum**.

### Why Contiguous?
Subarrays must be contiguous because:
- In painter's partition: A painter can't skip walls
- In book allocation: A student reads consecutive pages
- In split array: We're dividing a continuous sequence

## 🎨 Painter's Partition Problem

### Problem Statement

You have a wall divided into sections with different areas:
```
[10, 20, 30, 40]
```

**Given:**
- Each number represents units of area to paint
- Time to paint = area units (10 units = 10 time units)
- You have **K painters** (let's say K=2)

**Constraints:**
1. Each painter must get **at least one section**
2. Sections must be **contiguous** (consecutive)
3. Painters work **simultaneously** (parallel)

**Goal:** Minimize the maximum time taken by any painter

### Step-by-Step Breakdown

#### Example: Array = [10, 20, 30, 40], K = 2

**Possible Splits:**

| Split | Painter 1 | Painter 2 | Time P1 | Time P2 | Max Time |
|-------|-----------|-----------|---------|---------|----------|
| 1 | [10] | [20, 30, 40] | 10 | 90 | **90** |
| 2 | [10, 20] | [30, 40] | 30 | 70 | **70** |
| 3 | [10, 20, 30] | [40] | 60 | 40 | **60** ✓ |

**Answer: 60** (minimum of all maximum times)

#### Invalid Splits

❌ **Non-contiguous:**
```
Painter 1: [10, 30]  // Skipping 20
Painter 2: [20, 40]
```

❌ **Empty allocation:**
```
Painter 1: [10, 20, 30, 40]
Painter 2: []  // No work assigned
```

### Visual Representation

```
Array: [10, 20, 30, 40]
        ↓   ↓   ↓   ↓
       P1  P1  P1  P2

Painter 1 paints: 10 + 20 + 30 = 60 units (60 time)
Painter 2 paints: 40 units (40 time)

Both start simultaneously:
Time 0  ──────────────────────────────────────────────────────→
P1:     [════════════════════════════════════════════════════] (60)
P2:     [═══════════════════════════] (40)
                                      ↑
                              P2 finishes here
                                                               ↑
                                                      P1 finishes here

Total time = max(60, 40) = 60
```

**Minimize** the **largest sum** among these subarrays.

## 🎯 The Min-Max Pattern

### What is Min-Max?

> **Minimize the Maximum** or **Maximize the Minimum**

This pattern appears when:
- You need to distribute resources
- You want balanced allocation
- You're optimizing the worst-case scenario

### Why Binary Search?

#### The Key Insight

If we can complete the task with a maximum limit of `mid`, we can also complete it with any value greater than `mid`.

**This creates a monotonic search space:**

```
Can we split with max_sum = X?

[False, False, False, True, True, True, True, ...]
                      ↑
                First True = Answer
```

#### Search Space

**Lower Bound:** `max(array)` 
- Why? Each element must go to at least one subarray
- Example: [10, 20, 30, 40] → minimum possible max = 40

**Upper Bound:** `sum(array)`
- Why? Worst case: one subarray contains everything
- Example: [10, 20, 30, 40] → maximum possible max = 100

```
Search Space: [40, 41, 42, ..., 100]
                ↑                 ↑
              min(answer)      max(answer)
```

## 🔧 Solution Approach

### Algorithm Steps

1. **Define Search Space**
   - `low = max(array)`
   - `high = sum(array)`

2. **Binary Search**
   - Calculate `mid = (low + high) // 2`
   - Check: "Can we split array into K parts with max sum ≤ mid?"

3. **Validation Function**
   - Greedily assign elements to subarrays
   - Count how many subarrays needed
   - If count ≤ K → possible with this mid

4. **Adjust Search**
   - If possible → try smaller max (move left)
   - If not possible → need larger max (move right)

### Greedy Validation Logic

```
For a given max_sum limit:
1. Start with first subarray
2. Keep adding elements while sum ≤ max_sum
3. When adding next element exceeds limit:
   - Start new subarray
   - Increment subarray count
4. Check if total subarrays ≤ K
```

def splitArray(nums, k):
    """
    Split array into k subarrays to minimize the largest sum.
    
    Args:
        nums: List of integers
        k: Number of subarrays to split into
    
    Returns:
        Minimum possible value of the largest sum
    
    Time Complexity: O(n * log(sum - max))
    Space Complexity: O(1)
    """
    
    def canSplit(max_sum):
        """
        Check if we can split array into ≤k subarrays 
        with each subarray sum ≤ max_sum
        
        Args:
            max_sum: Maximum allowed sum for any subarray
        
        Returns:
            True if split is possible, False otherwise
        """
        subarray_count = 1  # Start with first subarray
        current_sum = 0     # Sum of current subarray
        
        for num in nums:
            # Try adding current number to current subarray
            if current_sum + num <= max_sum:
                current_sum += num
            else:
                # Need to start new subarray
                subarray_count += 1
                current_sum = num  # Start new subarray with current num
                
                # If we need more than k subarrays, it's impossible
                if subarray_count > k:
                    return False
        
        return True  # Successfully split into ≤k subarrays
    
    # Define search space
    low = max(nums)   # Minimum possible answer (largest element)
    high = sum(nums)  # Maximum possible answer (all elements in one)
    answer = high     # Initialize answer to worst case
    
    # Binary search for the answer
    while low <= high:
        mid = (low + high) // 2
        
        # Check if we can split with max_sum = mid
        if canSplit(mid):
            answer = mid    # This works, but try to find smaller
            high = mid - 1  # Search in left half
        else:
            low = mid + 1   # Need larger max_sum, search right half
    
    return answer
